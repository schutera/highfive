# Nest-hole grid reference (issue #165)

# SUPERSEDED. The original hand-measured pixel boxes below were taken on a
# single 791x528 reference frame and described a 4-bee-type x 3-nest layout.
# The physical laser-cut block actually has 4 nests per bee type (a 4x4 grid),
# and ESP captures arrive at VGA (640x480) or UXGA (1600x1200) — never 791x528 —
# so absolute pixel boxes don't transfer.
#
# The live detector (`image-service/services/hole_detection.py`) finds holes
# dynamically with OpenCV HoughCircles and labels bee type *by measured
# diameter*. There is NO fixed fallback grid: an earlier one fabricated holes
# when detection failed, which on real captures produced confident wrong data,
# so it was removed. Below a circle quorum the detector now returns no detection.
# Real-capture fixtures + the recalibration plan live in
# `dev-tools/real_captures/README.md`.
#
# Bee types are ordered by ascending hole diameter (homepage BEE_TYPES):
#   2 mm black-masked < 3 mm resin < 6 mm leafcutter < 9 mm orchard.
